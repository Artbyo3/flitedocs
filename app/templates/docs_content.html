Quick Start

Create your first Flask application in just a few commands:

1

Initialize Project

Create a new Flask project with the default template:

flite init my-awesome-app
2

Navigate to Project

Move into your new project directory:

cd my-awesome-app
3

Install Dependencies

Install the required packages:

pip install -r requirements.txt
4

Run Development Server

Start your Flask application:

flite serve

‚ö° Commands

Flite provides several commands to help you manage your Flask applications efficiently.

flite init

Primary

Initialize a new Flask project with a chosen template.

flite init [project-name] [--template TEMPLATE]

Options:

  • --template Choose project template (web-app, api-only, full-stack)
  • --database Specify database type (sqlite, postgresql, mysql)
  • --auth Include authentication system

flite generate

Utility

Generate additional components for your existing project.

flite generate [component] [--name NAME]

Components:

  • model Create a new database model
  • view Generate a new view/route
  • template Create a new HTML template

flite serve

Development

Start the development server with hot reloading.

flite serve [--port PORT] [--host HOST]

Options:

  • --port Specify port number (default: 5000)
  • --host Set host address (default: localhost)
  • --debug Enable debug mode

Configuration File

Flite generates a comprehensive configuration file for your project:

config.py 
import os

class Config:
    SECRET_KEY = os.environ.get('SECRET_KEY') or 'dev-secret-key'
    SQLALCHEMY_DATABASE_URI = os.environ.get('DATABASE_URL') or 'sqlite:///app.db'
    SQLALCHEMY_TRACK_MODIFICATIONS = False
    
class DevelopmentConfig(Config):
    DEBUG = True
    
class ProductionConfig(Config):
    DEBUG = False

üîß Troubleshooting

Common issues and their solutions to help you get unstuck.

ImportError: No module named 'flite'

Solution: Make sure Flite is installed in your current Python environment. Try running pip install flite or check your virtual environment activation.

Port 5000 already in use

Solution: Use a different port with flite serve --port 5001 or stop the process using port 5000.

Database connection errors

Solution: Check your database configuration in config.py and ensure your database server is running.
